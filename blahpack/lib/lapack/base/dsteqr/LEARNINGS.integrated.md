# dsteqr: Translation Learnings

## Translation pitfalls

- The Fortran has a complex GOTO structure with labels 10, 30, 40, 60, 80,
  90, 110, 130, 140, 160, 190. Restructured to `while(true)` loops with
  `continue`/`break` for the QL and QR iteration blocks, and a labeled
  `outer` while loop for the block-finding outer loop.
- The `SIGN(R, G)` idiom: `Math.sign(g)` returns 0 for g=0, but Fortran
  `SIGN(R,G)` returns +R for G=0. Used `(Math.sign(g) || 1.0)` pattern
  to compute `g + SIGN(R, G)`.
- dswap is a BLAS routine (not LAPACK), so require path needs 4 levels up
  from `lib/lapack/base/dsteqr/lib/` to reach `lib/blas/base/dswap/lib/`.
- The Fortran deps file generated by `init_routine.py` missed dswap (BLAS)
  and the Fortran module dependencies (la_constants, la_xisnan) needed by
  dlassq. Had to manually add them.

## Dependency interface surprises

- dlascl takes 2D stride parameters (strideA1, strideA2) even when applied
  to 1D vectors (N=1 column). Pass strideA1=strideD, strideA2=0 for the
  unused second stride since j=0 is the only column.
- dlasr's (c, s) arrays use separate stride/offset parameters, same as the
  WORK array layout (c stored in WORK[l..m-1], s in WORK[N-1+l..N-1+m-1]).
- dlae2 and dlaev2 return objects {rt1, rt2} and {rt1, rt2, cs1, sn1}
  respectively. No array/offset output convention for these.
- dlartg returns {c, s, r} object.

## Automation opportunities

- The deps file generator (`init_routine.py`) should detect BLAS dependencies
  and include them automatically. Currently it only looks in the LAPACK SRC
  directory, missing routines like dswap.
- The deps file should also auto-include Fortran module dependencies like
  la_constants and la_xisnan when dlassq is in the dependency chain.

## Coverage gaps

- `anorm === 0` path (line 190-192): requires all elements in a non-trivial
  block (lend > l) to be zero, which is practically unreachable since the
  block-finding loop would have split such a block.
- Convergence failure (jtot >= nmaxit): requires a matrix that doesn't
  converge in 30*N iterations. Pathological matrices for this are hard to
  construct and not useful for testing.
- QR path 2x2 with COMPZ=0 (dlae2 in QR branch): requires a specific
  matrix structure where QR iteration converges to a 2x2 block with
  eigenvalues-only mode.
- Undo scaling in QR path: requires combining descending diagonal with
  extreme values. Added a test for ssfmax scaling in QR path.
- Final coverage: 94.38% line, 88.35% branch -- above targets.

## Complex number handling

- N/A: dsteqr is a real routine. No complex arithmetic involved.
