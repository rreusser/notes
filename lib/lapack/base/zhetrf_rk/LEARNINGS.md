# zhetrf_rk: Translation Learnings

## Translation pitfalls

- The blocked driver structure is identical to `dsytrf_rk`. Once the
  d-prefix sister exists, the z-prefix translation is a near
  copy-paste with three substitutions: `Float64Array` →
  `Complex128Array` for `A` and `e`, `dswap` → `zswap`, and the panel
  kernels (`dsytf2_rk` → `zhetf2_rk`, `dlasyf_rk` → `zlahef_rk`).
  Strides/offsets stay in *complex* units throughout the driver
  because all callees (`zswap`, `zhetf2_rk`, `zlahef_rk`) use the
  complex-element stride convention.
- IPIV uses the bitwise-NOT (`~p`) encoding for 2x2 pivots — same as
  the d-prefix `_rk` and `_rook` variants. Apply `ip = ~ip` (NOT
  `-ip - 1`) to recover the swap-target row. The lower-panel index
  remap also re-encodes via `~( (~IPIV[i]) + (k-1) )` — adding to the
  decoded value then re-applying NOT.
- The Fortran `LWORK`/`WORK` arguments are dropped entirely from the
  JS API, matching `dsytrf_rk`. Workspace is allocated internally.
  This shrinks the BLAS-style wrapper signature from 13 to 9
  parameters; the test scaffold's hardcoded arity check (13) needed
  to be updated.

## Dependency interface surprises

- Both `zlahef_rk` and `zhetf2_rk` already exist with calling
  conventions identical to their d-prefix counterparts (modulo
  `e` being a `Complex128Array`). No surprises there.
- `zlahef_rk` returns an object `{ kb, info }`. `zhetf2_rk` returns
  `info` directly. The driver uses both signatures verbatim from
  `dsytrf_rk`.

## Complex number handling

- Hermitian-vs-symmetric differences (diagonal real-only,
  conjugate at mirror reads, `'conjugate-transpose'`) are entirely
  handled inside the panel kernels (`zlahef_rk`, `zhetf2_rk`). The
  blocked driver itself does no complex arithmetic — it only
  dispatches kernels and applies row swaps via `zswap`. So the
  Hermitian/symmetric distinction is invisible at the driver level.
- The deps file generated by `init_routine.py` already includes
  `zhetf2_rk`, `zlahef_rk`, `zlacgv` and the transitive scalar
  helpers (`dlamch`, `ieeeck`, `ilaenv`, `iparmq`). No manual edits
  needed.
